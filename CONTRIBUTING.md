# Contributing to FlexiCalendar

Thanks for taking the time to contribute! This project aims to provide a flexible, accessible calendar component for the React ecosystem. Contributions of all sizes are welcome.

Please make sure you have read and follow our [Code of Conduct](CODE_OF_CONDUCT.md).

## Ways to Help

- Report bugs or usability issues
- Suggest new features or improvements to the API
- Improve documentation, examples, or demos
- Tackle issues tagged with `good first issue`

## Development Setup

1. Fork the repository and clone your fork.
2. Install dependencies:
   ```bash
   npm install
   ```
3. Run the playground locally:
   ```bash
   npm run dev
   ```
4. Write tests or add examples where appropriate.
5. Ensure `npm run lint` and `npm run build` succeed before opening a pull request.

## Coding Guidelines

- Follow the existing project structure in `src/` (components, hooks, utils, styles).
- Prefer readable, well-factored code with minimal abstractions.
- Add succinct comments only when necessary to explain non-obvious logic.
- Keep the codebase ASCII-only unless dealing with content that explicitly requires Unicode.
- Run `npm run lint` before submitting your changes.
- Keep pull requests focused and scoped to a single concern.

## Pull Requests

- Create a feature branch for your work.
- Reference related issues in your PR description.
- Include screenshots or recordings for UI changes when helpful.
- Describe testing steps and edge cases covered.
- Review the pull request template checklist before submitting.

## Community

Questions, ideas, or feedback? Open a discussion or start a new issue and we will respond as quickly as possible. Thank you for helping improve FlexiCalendar!
